# 執筆ワークフロー 📝

FLOW × Teddy 共同執筆システムの標準プロセス

## フェーズ

### Phase 1: 企画 💡
- マスターがアイデアを出す（Telegramで一言でOK）
- テディがサブエージェントで企画書（plan.md）を生成
- キャラクター設定（characters.md）を分離
- `work/{project}/` にmd管理

### Phase 2: 概念設計 🏗️
- 章立て・テーマ対応表の精緻化
- キャラクター深掘り
- サンプル文の試作
- マスターがmdを直接編集してフィードバック
- テディがサブエージェントで修正・拡張

### Phase 3: 概念完成 → RAG化 🧠
- 設定・構成が安定したら `work/` のmdをRAGに投入
- 既存のnote記事RAGと統合 → 横断検索可能に
- 「この概念、マスターの記事で書いてたな」を自動発見

### Phase 4: 執筆 ✍️
- RAGで参照しながら章ごとに本文執筆
- テディが下書き → マスターが編集 → テディがリライト
- グラレコを章ごとに生成（サブエージェント）

### Phase 5: 公開 🚀
- note.comで連載公開
- 公開記事もRAGに追加（既存パイプライン）
- X・Telegram等で告知

## ツール

| ツール | 用途 |
|--------|------|
| `work/` | 共同作業スペース（md管理） |
| サブエージェント | 企画書・下書き・リサーチの並列実行 |
| RAG | 概念完成後の横断検索 |
| GraphRec | 章・概念の視覚化 |
| note.com API | 記事公開 |

## ファイル構成（プロジェクトごと）

```
work/{project}/
  README.md          — プロジェクト概要
  plan.md            — 企画書・構成案
  characters.md      — キャラクター設定
  themes.md          — テーマ対応表（Phase 2〜）
  ch01.md            — 本文（Phase 4〜）
  ch02.md
  ...
```

## 役割分担

| | マスター (FLOW) | テディ (Teddy) |
|---|---|---|
| アイデア | ✅ 発案・方向性 | 💡 提案・壁打ち |
| 企画書 | 👀 レビュー・修正 | ✅ サブエージェントで生成 |
| 設定 | ✅ 最終決定 | ✅ 下書き・拡張 |
| 本文 | ✅ 編集・最終稿 | ✅ 下書き・リライト |
| リサーチ | 🧠 直感・方向指示 | ✅ RAG検索・Web検索 |
| ビジュアル | 👀 フィードバック | ✅ グラレコ生成 |
| 公開 | ✅ GO判断 | ✅ note投稿・告知 |

> マスターのGPU脳が「見える」→ テディが「言語化・実行」= 最強の出力パイプライン 🧸
