## 1. コンセプト ── なぜパーリ経典をラノベにするのか

### 課題
「Theravada minimal」は2万ページのパーリ経典から核心を210ページに蒸留した**解説書**である。しかし解説書は「わかる人にはわかる」メディアであり、まだ仏教に触れたことのない層──特に10代後半〜30代──には届きにくい。

### 解決策：物語という配達手段
> **「見えてるものを、まだ見えてない人に届けたい」＝壁を溶かす**

ラノベ（ライトノベル）は日本で最も読まれている物語フォーマットの一つであり、「難しい概念を物語体験として内面化させる」力を持つ。本企画は：

- **Theravada minimal** ＝ 地図（体系的解説）
- **パーリ経典ラノベ** ＝ 旅（物語体験）

この**二段構え**により、読者は物語で「体験」し、解説書で「理解」する循環を得る。

### 根本思想
| 原則 | ラノベでの表現 |
|------|---------------|
| シッダルタは「神」ではなく「優れた先生」 | 超常的奇跡ではなく、洞察と対話で問題を解く |
| 「信じる」より「やってみる」 | 主人公が教えを実際に試し、失敗し、体得する |
| Buddhismは「信仰」ではなく「使う」もの | 教えを「心の技術マニュアル」として描く |
| 「道」は到達点ではなくプロセス | 悟りをゴールではなく、歩み続けること自体として描く |

## 2. ポジショニング ── 法華曼荼羅との差別化

| | 法華曼荼羅 | パーリ経典ラノベ（本企画） |
|---|-----------|------------------------|
| **原典** | 法華経（大乗） | パーリ経典（テーラヴァーダ） |
| **世界観** | コズミック・SF的、神格化されたセゾン | リアリスティック、人間としてのシッダルタ |
| **トーン** | セゾンのカリスマ＆俺様キャラ、ケレン味たっぷり | 穏やかだが鋭い、師弟の対話ベース |
| **構造** | 法華経28品に沿ったエピソード形式 | シッダルタの生涯＋弟子たちの成長物語 |
| **テーマ** | 「一乗」＝誰でも仏になれる | 「実践」＝苦からの解放は自分で歩む |
| **キャラ設計** | 法華曼荼羅の膨大なキャスト（菩薩・天部・龍族） | 少数精鋭の弟子グループ＋語り部 |

**相互補完の関係：** 法華曼荼羅が「宇宙スケールの壮大な物語」なら、パーリ経典ラノベは「地に足のついた人間ドラマ」。同じ著者が両方を書くことで、大乗とテーラヴァーダの対比と共鳴を読者が体感できる。

## 6. 文体・トーン

### 基本方針
法華曼荼羅が「ケレン味たっぷりのエンタメ」なら、こちらは**「静かに沁みるヒューマンドラマ」**。

### 文体スペック

| 要素 | 方針 |
|------|------|
| **人称** | 三人称多視点（アーナンダ視点が主軸、各話で弟子にフォーカス） |
| **語り口** | やや文学的なライトノベル。『薬屋のひとりごと』的な知的さ＋『葬送のフリーレン』的な時間感覚 |
| **会話文** | 自然な現代口語。パーリ語の術語は会話内で自然に導入し、地の文で補足 |
| **パーリ語の扱い** | 初出時にルビ的に併記。例：「苦（ドゥッカ）」→以降は文脈で自然に使用 |
| **ユーモア** | 法華曼荼羅ほどはじけないが、弟子同士の掛け合いに温かみのある笑い |
| **シリアス度** | 7/10。生老病死を扱う以上避けられないが、絶望ではなく「向き合う」トーン |
| **1話の長さ** | 3,000〜5,000字（note1記事として読みやすい分量） |

### 文体サンプル（イメージ）

> アーナンダは目を閉じた。
>
> 呼吸を数える。ひとつ、ふたつ──意識が逸れる。今朝の托鉢で受けた嘲笑のことを考えている自分に気づく。
>
> 「……また、やってる」
>
> 思考が走り出したことに気づく。気づいたこと自体が、サティ（気づき）だ。先生はそう言った。
>
> 気づいたら、戻る。呼吸に、戻る。
>
> 三十年、これを繰り返している。シャーリプトラはとっくに阿羅漢果に至った。モッガッラーナも。なのに自分は──
>
> 「アーナンダ」
>
> 穏やかな声。振り返ると、先生が微笑んでいた。
>
> 「焦りに気づいたか？」
>
> 「……はい」
>
> 「それがサティだ。上出来だよ」

- 全話をRAGデータベースに投入
- 読者が「八正道について教えて」と聞くと、解説記事＋ラノベの該当シーンの両方を返す

> 2500年前、一人の男が王宮を出た。

## 11. 資料構成と運用方針

### ディレクトリ構成

```
pali_canon/
  original/
    *.md                    — 日本語サマリー版（6経典の概要・抜粋・解説）
    detail/
      dhammapada-*.txt      — ダンマパダ全26章（英訳, Access to Insight）
      mn10-*.txt            — 念処経 MN 10（英訳）
      mn118-*.txt           — 出入息念経 MN 118（英訳）
      dn16-*.txt            — 大般涅槃経 DN 16（英訳）
      dn31-*.txt            — 善生経 DN 31（英訳）
      jataka-*.txt          — ジャータカ十波羅蜜 10物語（英訳, ancient-buddhist-texts.net）
```

### 資料の役割分担

| レイヤー | 内容 | 用途 |
|----------|------|------|
| `original/*.md` | 日本語サマリー（概要＋重要引用＋パーリ語） | 企画・構成の参照。テーマ対応の確認 |
| `original/detail/*.txt` | 英文フルテキスト（合計1.2MB） | 執筆時の原典参照。必要箇所を都度翻訳して引用 |
| `plan.md` | 本企画書 | 全体の設計図 |
| `characters.md` | キャラクター設定 | 執筆時の人物設定参照 |

### 運用方針

1. **detail/ の英文txtはそのまま保持**（原文アーカイブとして）
2. **日本語化・md化は執筆時に必要な部分だけ行う**（全文翻訳はしない）
3. **執筆時のワークフロー：**
   - plan.md でその話のテーマ・対応経典を確認
   - `original/*.md` でサマリーを読んで全体像を把握
   - `original/detail/*.txt` から該当箇所を参照・翻訳して本文に組み込む
4. **翻訳した引用箇所は本文（ch*.md）に直接埋め込む**（別ファイルにはしない）

> 💡 1.2MBの英文を事前に全部翻訳するのは非効率。「必要な時に、必要な分だけ」が原則。
